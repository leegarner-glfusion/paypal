PayPal Plugin

PayPal Plugin Configuration Options

Main PayPal Settings
Option Description
Administrator' Email Address Enter the email addresses to which confirmation email messages should be sent. If this is empty, the site email address will be used.
Currency This is the currency in which you want all transactions to occur. Your Paypal account must have the choosen currency enabled (on the Paypal website. See: Selling Preferences->Payment Receiving Preferences).

This plugin currently requires you to have only one currency enabled in your Paypal account. All the currencies listed in ISO 4217 are available for selection, If the selected currency is not supported by a payment gateway such as PayPal or Amazon the gateway will be disabled.

Anonymous users can buy? Select "Yes" if you allow anonymous users to buy items from your site. Select "No" to allow only registered usrs to purchase.

See "Attach files to anonymous buyers' email" below if you allow anonymous purchases of downloadable files.

Email User upon purchase? Specify whether "on purchase" emails will be sent. Be sure to edit the files templates/purchase_email_subject.txt and templates/purchase_email_message.txt to customize the email message.
You should disable the *_attach options below if you are distributing large files or it is likely that a purchaser will be buying many medium files otherwise the email will become too large and will likely never be received.
Attach files to user's email message? Indicate whether the purchase confirmation email to registered users (only) includes the file to download.

See "Attach files to anonymous buyers' email" to set this function for anonymous purchasers.

Email anonymous users upon purchase? Select whether to email anonymous users upon purchase. See "Email Users upon purchase" for details.
Attach files to anonymous buyers' email? Select whether to attach the purchased files to emails sent to anonymous users.

This should be "Yes" if you allow anonymous purchases of downloadable items, since this is currently the only way to distribute files to these buyers unless you send them via another method.
Notify administrators of purchases? Select if and when the site administrator should receive an email notification of a purchase. The options are:

Never
No notifications will be sent to the administrator.
Physical Items Only
A notification will be sent if the order contains a physical item. If the order is for only virtual items, such as downloadable files, then no email is sent.
Always
An email is sent for every purchase.

If your site sells physical items that require further action on your part, such as shipping, then you are encouraged to enable notifications for at least Physical Items Only.
Add to main menu? Select "Yes" to have the "Product" menu option added to the plugin menu. Select "No" if you wish to manually add this plugin to another menu.
Default sort order for product display Select the default sort order for products shown on the main product list.
Max products displayed per page Set the maximum number of products to display on a single page.
Category Columns Set the number of columns in which to display categories on the product display page.
Use Internal CSS Menus? Select "Yes" to use the internal CSS-based tabbed menu. Select "No" to use the built-in glFusion menu style.
Enable Comments? Select whether you'd like to enable user comments for products.
Enable Product Ratings? Select whether you'd like to enable user ratings for products.
Anonymous can rate products? Select whether you'd like to allow anonymous users to rate products. If this is "No" then only logged-in users can give ratings.
Display glFusion Blocks Select whether to display left blocks, right blocks, both, or neither when the product catalog is displayed.
Enable shopping cart? Select whether the internal shopping cart should be enabled. If this is not enabled, there will be no way for a buyer to purchase items with various attributes, as those items must go through the shopping cart and checkout process.
Image and Path Settings
Option Description
Max. Product Images Enter the maximum number of images to be associated with a single product. This determines how many upload fields are shown on the product form.
Max. Image Size Enter the maximum size, in bytes, for uploaded images. Only PayPal administrators can update product records, so this is simply a sanity check. This number should be less than or equal to the upload_max_filesize setting for PHP.
Max. Thumbnail Dimension Enter the maximum size, in pixels, for thumbnail images. The thumbnail will be sized so that its longest side does not exceed this value while preservint the aspect ratio.
Max. Image Width Enter the maximum width, in pixels, for uploaded images. The display version of the image, shown when the thumbnail is clicked, will be sized so that its dimensions do not exceed this width or the Max. Image Height height, while preserving the aspect ratio.
Max. Image Height Enter the maximum height, in pixels, for uploaded images. The display version of the image, shown when the thumbnail is clicked, will be sized so that its dimensions do not exceed this height or the Max. Image Width widgh, while preserving the aspect ratio.
Full Path to Downloadable Files This is the full path, including trailing slash, where files will be stored for later download by the buyer.

The default path is created under the private/data/paypal directory during plugin installation. This path may be changed, but must be readable by your web server. If you wish to be able to upload product files via the product form, then it must also be writable.
Max size for downloadable files This is the maximum size, in MB, of files that will be downloadable by buyers. This number actually limits the size of file uploads, not downloads.
Temporary Working Directory Enter the full path to a working directory to be used during the creation of encrypted buttons. The web server must be able to create and delete files in this directory.

When buttons are encrypted, several temporary files are created and then deleted. These do not contain sensitive information, but this directory should still be fairly private.

New Product Defaults
These options set the default values for new products and can be overridden for each product.
Option Description
Product Enabled? If "Yes", then the "Enabled" checkbox will be checked automatically for new products.
Product is Taxable? If "Yes", then the "Taxable" checkbox will be checked automatically for new products. This checkbox only has an effect if you've set up tax schedules in your PayPal profile.
Product Featured? If "Yes", then the "Featured" checkbox will be checked automatically for new products.
Track Qty On Hand? Select "Yes" to have the plugin track the quantity on hand of an item, by default. The administrator can update the quantity by editing the product, and the quantity will be reduced as purchases are made (to a minimum of zero).
Action when Qty On Hand = 0 Select the action to be taken when the quantity on hand reaches zero. Options are:
  • Allow Backordering: Continue to allow sales of the product.
  • Hide from Catalog: Hide the product from the catalog so it cannot be seen or purchased.
  • Keep in Catalog, Prevent Sales: Continue to display the product in the catalog but remove the purchase links.
Product Detail Template Version

Select the default template version to use for the Product Detail views. Two template versions are currently included: the previous template and a new version that emphasizes product images. This is currently a global setting and affects all products but a future version may include per-product selections.

You may add your own templates by creating a directory under private/paypal/templates/detail and creating product_detail.thtml and product_detail_attrib.thtml. Both files are required (this may change later).
Block Settings
Option Description
Number of products in Random block This is the number of products that will be shown in the "Random Product" block. If this number is less than one, then one product will be shown.
Number of products in Featured block This is the number of products that will be shown in the "Featured Product" block. If this number is less than one, then one product will be shown.
Number of products in Popular block This is the number of products that will be shown in the "Popular Products" block. If this number is less than one, then one product will be shown.
Debugging
Program Debug? Select "Yes" to have messages sent to the error log when the plugin performs various actions. This may be useful to debug why the plugin isn't behaving as you expect.
Debug IPN Messages? Select "Yes" to have the complete Instant Payment Notification (IPN) message from Paypal logged to your site's error log. This can be useful to troubleshoot difficulty with handling the messages.
Checkout Address Collection
Option Description
Street Address At checkout, the Street Address field can be Required, Optional, or Not Available for the Billing and Shipping addresses (whichever are used).
City At checkout, the City field can be Required, Optional, or Not Available for the Billing and Shipping addresses (whichever are used).
State At checkout, the State field can be Required, Optional, or Not Available for the Billing and Shipping addresses (whichever are used).
Country At checkout, the Country field can be Required, Optional, or Not Available for the Billing and Shipping addresses (whichever are used).
Postal Code At checkout, the Postal Code field can be Required, Optional, or Not Available for the Billing and Shipping addresses (whichever are used).
Shop Information
Option Description
Shop Name Enter the name of your store. During installation this was set to the site name but can be changed here. This will be displayed on remittance coupons and receipts.
Shop Address Enter the remittance address for your shop. This value is optional, but should be provided if you accept payments by check.
